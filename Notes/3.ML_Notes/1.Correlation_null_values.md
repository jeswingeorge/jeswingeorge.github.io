# Correlation NULL values

While taking part in a Kaggle playground competition, found this issue

Correlation coefficient were null for these numerical features having discrete values: 

```
['Time_spent_Alone',
 'Social_event_attendance',
 'Going_outside',
 'Friends_circle_size',
 'Post_frequency']  
 ```

![](images\1.PNG)


The correlation coefficient (especially Pearson's) can be misleading or even return `nan` (null) when:

1. **Discrete or Categorical Data**:  
   Pearson's correlation assumes continuous, normally distributed variables. Your features are discrete (often ordinal or count data), which can violate these assumptions.

2. **Low Variance or Constant Values**:  
   If any feature has little or no variance (e.g., all values are the same or nearly the same), the correlation calculation will return `nan` because the denominator becomes zero.

3. **Small Number of Unique Values**:  
   If a variable has very few unique values (e.g., only 2 or 3), Pearson's correlation is not reliable.

**What to do?**  
- For discrete/ordinal features, use **Spearman's rank correlation** (`spearmanr`), which works with ranks and is more robust for such data.
- Always check for constant columns or columns with very low variance.

**Example:**
````python
from scipy.stats import spearmanr

for col in ['Time_spent_Alone', 'Social_event_attendance', 'Going_outside', 'Friends_circle_size', 'Post_frequency']:
    corr, _ = spearmanr(df[col], df['Time_spent_Alone'])
    print(f"Spearman correlation between Time_spent_Alone and {col}: {corr}")
````

The null correlation is likely due to the discrete nature and/or low variance of your features. Use Spearman's correlation for better insight. 

```
> spearmanr(df['Social_event_attendance'], df['Time_spent_Alone']).correlation
> nan
```

If both Pearson and Spearman return null, it means correlation cannot be computed for those variables in your dataset, likely due to lack of variability. This is a data property, not a limitation of the correlation methods.
